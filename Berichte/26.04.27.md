# Linux Permissions

## Vorbereitung der Übungsumgebung
Laborübung Vorbereiten:
Mit nano linux.sh anlegen und einfügen, dannach ausführen

```ssh
#!/bin/bash

# Gruppen anlegen
groupadd music
groupadd wrestle
groupadd emperors

# User anlegen und Gruppen zuweisen
useradd -m -G music elvis
useradd -m -G music,wrestle blondie
useradd -m -G wrestle hogan
useradd -m -G emperors nero

# Passwörter auf 'kali' setzen
echo "elvis:kali" | chpasswd
echo "blondie:kali" | chpasswd
echo "hogan:kali" | chpasswd
echo "nero:kali" | chpasswd

# Lab-Struktur vorbereiten
mkdir -p /tmp/lab/gigs /tmp/lab/drafts
echo "Draft" > /tmp/lab/drafts/lyrics.txt
echo "Plans" > /tmp/lab/wrestle_plans.txt
chown -R root:root /tmp/lab
chmod -R 755 /tmp/lab
```

## Übung (Analyse und Symbolic Mode)
### Rechte auslesen:
```ssh
──(kali㉿kali)-[~]
└─$ ls -l /tmp/lab/wrestle_plans.txt
-rwxr-xr-x 1 root root 6 Apr 27 05:56 /tmp/lab/wrestle_plans.txt
```
- Stelle 2-4 (rwx): Rechte für den Besitzer. Er darf lesen (read), schreiben (write) und ausführen (execute).
- Stelle 5-7 (r-x): Rechte für die Gruppe (Group). Sie darf lesen und ausführen, aber nicht schreiben (gekennzeichnet durch das -).
- Stelle 8-10 (r-x): Rechte für Alle anderen (Others). Auch sie dürfen nur lesen und ausführen.

**Der Besitzer (root)**   
Das erste „root“ nennt dir den User Owner. In diesem Fall gehört die Datei dem Systemadministrator (Root).   
**Die Gruppe (root)**   
Das zweite „root“ nennt den Group Owner. Die Datei ist der Gruppe „root“ zugeordnet.

### Private Files
File erstellen und rechte geben/nehmen:  
```ssh
touch notes.txt
chmod o-r notes.txt
```

### Group Sharing
u=rwx: User darf alles   
g=rw: Group darf lesen/schreiben   
o=: Others haben gar keine Rechte   
mit Beistrichen trennen:   
```ssh
chmod u=rwx,g=rw,o= notes.txt
```

### Skript-Vorbereitung   
mit user u und gruppe g das ausführen x erlauben.   
mit touch erstellen und dann ausführen:   
```ssh
touch myscript.sh
chmod ug+x myscript.sh
```

### Kollektive Änderung
Datei erstellen und group und other die rechte w&x enziehen: 
```ssh
touch data.txt
chmod go-wx data.txt
```

## Übung (Ownership und Gruppenkollaboration)
### Blondie in sudoers hinzufügen
1. sudo usermod -aG sudo blondie

### Group Change
2. touch song.txt
3. sudo cp song.txt /tmp/lab
4. sudo chgrp music /tmp/lab/song.txt

### Unterschied
su <user> (Substitute User): Wechselt den User, behält aber deine aktuelle Umgebung bei   
su - <user> (Login Shell): Wechselt den User und lädt dessen komplette Umgebung (Home-Verzeichnis, Pfade, Aliase).

### User Transfer
```ssh
chown elvis /tmp/lab/song.txt
```
elvis ist nun User Owner der datei song.txt
