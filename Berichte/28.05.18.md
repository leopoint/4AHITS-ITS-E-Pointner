# Übung (Directory Permissions und Octal Notation)
## Oktale Umwandlung

**rwxr-xr-x**   
Besitzer: $4+2+1 = 7$  
Gruppe: $4+0+1 = 5$  
Andere: $4+0+1 = 5$  
Befehl: chmod 755 <datei/ordner> 
 
**rw-rw----**   
Besitzer: $4+2+0 = 6$  
Gruppe: $4+2+0 = 6$  
Andere: $0+0+0 = 0$  
Befehl: chmod 660 <datei/ordner> 
 
**rwx------**   
Besitzer: $4+2+1 = 7$  
Gruppe: $0+0+0 = 0$  
Andere: $0+0+0 = 0$  
Befehl: chmod 700 <datei/ordner> 

## Pub-Verzeichnis
```ssh
mkdir -p /home/kali/pub
chmod 751 /home/kali/pub
```

## Recursive Lockdown
```ssh
chown -R :music /tmp/lab/drafts/
sudo chmod -R g+rw,o-rwx /tmp/lab/drafts/
```

## Search Permission
```ssh
chmod o+x,o-r /home/kali
```
Alle anderen Verzeichnisse sind nach diesem Befehl für andere User unsichtbar aber /home/kali/pub funktioniert immer noch.

## Analyse  
Durch den Befehl `660` haben alle Ordner nur noch die Rechte `rw-`. Es fehlt das `x`.  

Bei Ordnern bedeutet das `x` aber nicht "Ausführen" wie bei einem Programm, sondern **"Betreten" (Search Permission)**.  

* **Das Problem:** Ohne das `x`-Bit ist der Ordner quasi "abgeschlossen". Man kann zwar sehen, was drin ist (wenn `r` erlaubt ist), aber man darf nicht hineingehen.  
* **Fehlender Permission-Typ:** Es fehlt das **Execute-Bit (`x`)** für Verzeichnisse.  
  
# Übung (Webserver Directory Permissions)
## Teil 1: other/read weggenommen  
cd /var/www möglich? Ja, weil das x-Bit (Betreten) noch da ist.  
Inhalt anzeigen (ls) möglich? Nein, ohne r siehst man nichts.  
Wechseln in html möglich? Ja, wenn man den Namen kennt, kommt man mit cd html durch.  

## Teil 2: other/read da, other/x weg 
cd /var/www möglich? Nein, ohne x ist der Ordner für cd abgeschlossen.  
Inhalt anzeigen (ls) möglich? man sieht zwar die Dateinamen, aber nur mit Fehlermeldungen (Fragezeichen statt Details), weil das System nicht tiefer reinschauen darf.  
Zugriff auf Unterordner (html)? Komplett blockiert. Da der Weg durch /var/www gesperrt ist.
