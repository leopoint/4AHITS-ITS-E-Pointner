## Übung (netcat port scan)
```sh
nc -zv 192.168.14.52 1-30
```
-z: prüfen, ob Ports offen sind, ohne Daten zu senden

-v: zeigt Ergebnisse an

1-30: Ports 1 bis 30 automatisch durchscannen

## Übung (nmap port scan)
1. Normaler nmap Port scan:  
   nmap 192.168.14.52  
   **Analyse:**
   
   | Ausgabe | Erklärung |
   |---|---|
   | Nmap done: 1 IP address (1 host up) scanned in 0.45 seconds | In 0.45 Sekunden 1 Host gefunden |
   | 21/tcp | FTP übertägt passwörter unverschlüsselt |
   | 23/tcp | Telnet übertägt alles im Klartext und jeder im Netzwerk kann alles mitlesen |
   | 3306/tcp | MySQL angreifer können Datenbanken auslesen und Tabellen manipulieren |

 3. Experimentieren mit verschiedenen Optionen   
    -p-: gibt alles ports aus da nmap es auf eine gewisse Anzahl begrenzt (dauert dann auch länger)
    
 4. Bei zb manchen Firmennetzwerken erden pings blockiert -> Host down -> namp scannt nicht   
    -Pn: Bei -pn sagt man nmap das er UP ist dadurch ermöglicht es den nmap scan
    
 5. -sV: gibt die exakte Version des Dienstes aus   
    Ermöglicht Sicherheitslücken aus zb veralteten Versionen im Internet zu finden

 6. -O: gibt das Betriebssystem aus -> wahl von geeigneten Exploits   
 7. -A: kombination aus -sV und -O in einem einzigen Befehl

 8.

| Timing-Level | Name        | Geschwindigkeit     | Einsatzgebiet |   
|--------------|-------------|----------------------|----------------|   
| **T0**       | Paranoid    | Sehr langsam         | Maximal stealth, IDS umgehen |   
| **T1**       | Sneaky      | Langsam              | Unauffällige Scans, Firewalls umgehen |   
| **T2**       | Polite      | Moderat              | Netzwerk nicht belasten, vorsichtige Scans |   
| **T3**       | Normal      | Standardgeschwindigkeit | Standard für typische Scans |   
| **T4**       | Aggressive  | Schnell              | Lokale Netze, Pentests, Trainingsumgebungen |   
| **T5**       | Insane      | Sehr schnell         | Nur stabile Netzwerke/Labors, kann ungenau sein |

 9. Wichtig für gute Dokumentation
      
| Option | Format | Beschreibung |        
|--------|---------|--------------|      
| **-oN** | Normal (Text) | Lesbares Standardformat, wie im Terminal |       
| **-oG** | Greppable | Für Skripte gedacht, leicht mit grep/awk zu verarbeiten |     
| **-oX** | XML | Maschinell auswertbar, kompatibel mit vielen Tools |   
| **-oA** | All | Erstellt **alle drei** Ausgaben gleichzeitig (txt, grep, xml) |   

 10. --reason: gibt an ob der Port open, closed, filtered ist
 11. wegen fehlender Zeit nicht fertig geworden


## Übung (nmap cheat sheet)
**Grundlagen**  
nmap [Optionen] [Ziel]  

**Ziele**  
orf.at oder IP - Einzelnes Ziel  
192.168.1.0/24 - Subnetz  
-iL datei.txt - Ziele aus Datei  

**Host-Erkennung**  
-sn - Nur Host-Erkennung (kein Portscan)  
-Pn - Hosts als online behandeln  

**Scan-Techniken**  
-sS - SYN-Scan (Standard, schnell)  
-sT - Connect-Scan (voller Handshake)  
-sU - UDP-Scan  
-sV - Versionserkennung  
-O - Betriebssystemerkennung  

**Ports**  
-p 80,443 - Spezifische Ports  
-p- - Alle Ports (1-65535)  

**Ausgabe**  
-v - Ausführlichkeit erhöhen
